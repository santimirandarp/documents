AHA! Floats are removed from the normal flow of the document. The browser measures ul's height before throwing in the children, then height = 0 as default height for blocks is auto. It also explains the absence of bgcolor.

But why are the items displayed at all?

The default value for overflow is 'visible', so we are actually seeing the content-overflow.

But does the element has a width?

We can test it by assigning a height:

The ul has 100% width, as it is a block element. If height is 0 it is not 'seen' by the browser, and we are left with only three inline elements (list items).

We may try to add anchors and some padding though it must suit the container's height. Not the easy way if we need to re-set the height. Also, we know that height:auto is not an option.
