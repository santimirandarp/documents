Networks

We talked about some ways, namely protocols, in which two computers - in a network - can exchange data. Let's now see how to set up a network.(Yes, not the what, but the how)

Types

Knowing the types of network simplifies the problem. We can classify networks depending on the physical area they span, and whether they are wired or wireless.

  • pan. Devices in a pan can be connected via bluetooth, ir, nfs; or by cable.
  • lan, wlan. Offices, houses. Devices connect to the network through a switch by cable or to a router using a wifi card. lan is wired-only; wlan is wireless instead. Nowadays, the router usually includes the switch. The switch-part are the ports. Soho networks are an example of a lan/wlan.

    man, wan. Nowadays connection is usually by fiber cables. World Wide Web is a wan. Devices in this network are connected using fiber cables. We reach the wan from the lan if the Modem is connected to the wan.

    A lan does not need a wan, but can connect to it.

    Hardware

    1. Modem: Connects (data) our network with the external world (internet) via DSL, Satellite, Cable or Fiber.
    2. Router: the one who draws the route. This route is the path the data follows, from a device to a device. It connects different networks. The way computers find each other is specified in the Internet Protocol (IP). AFAIK any connection between 2 networks needs a router.
    3. Firewall
    4. Switch: connects cabled devices to a network. There are unmanaged and managed switches. The latter ones can be programmed for specific tasks. The switch supports particular speeds.
    5. Wireless Access Point or WAP (optional): connects wifi devices to a network. It's linked to the switch.
    6. Computer

    Modem

    The modem is usually connected to a Fiber Cable or to a Satellite on one end. On the other end it connects to the Router or Switch.

    The word modem comes from modulate/demodulate. It could be electric to sound (DSL) and viceversa.

    WAP

    Wireless network devices 802.11a, 802.11b, 802.11g, 802.11n. Most of them are compatible to each other (only a is useless), so buying any network card should work. We should pick up the best speed match though. n-type spans a wide area and is quick.

    Firewall

    A firewall config may not be necessary because this is done in the router.

    A firewall stops the fire, in this case, dangerous data. Host-based firewalls are the ones set up on a computer, and protect only that computer. Windows comes with firewall and also 3rd parties can be installed. Antivirus might come with firewall. Network-based firewall rules are applied to the entire network. This is probably set in the switch or Router, by tweaking its config using a web-gui.

    Cabling

    Example:SOHO

    soho networks are small networks; like a lan at home. For this kind of networks we can buy a particular type of Routers. Enterprise routers only route traffic over/in/out the lan. Soho routers also include many devices apart from the router: Switch (those phone plugs on the back), DNS server, DHCP server, Firewall server, and Port forwarding.

    We can configure a soho Router using a web browser, by typing the router's IP in the URL bar. This is not the way to set up an enterprise router.

    When a device connects to a network, it either has a static or a dynamic IP. Normally, we want the DHCP server to assign all the network parameters automatically. For that, we configure an IP range on the DHCP server.

    If there is a server on the lan, we want a static ip. Let's say someone in a different part of the world types our domain in a browser. They get the domain's IP from a DNS server. The request lands to the router at a particular port. This depends on the type of request. We need to link the IP of our lan server to the specific port. Web requests arrive at port 80, ftp at 21, etc etc. We can look them up online.

    Then, in the port forward section, we write our server IP and the port it will be listening to. Example 192.168.0.1:80

    If the server IP is not static, then the redirection will occur either to another device, or to nowhere, and the request will fail. So, for servers, we need a static IP.
