Some Data Transfer protocols

A pre-condition to any of this methods, is two devices connected in a network. All devices with this capability have an IP (internet protocol). With this condition fulfilled, we can exchange data through the following protocols (non-exhaustive list):

  • http, ftp
  • https, sftp
  • pop3, smtp, imap

  • ssh
  • scp
  • sshfs
  • nfs, samba, giver

sftp, scp, sshfs are all ssh-dependent, i.e they work behind the same security principles than ssh.

pop3, smtp, imap are all internet, mail specific protocols

FTP, FTPS, HTTP, HTTPS

These are all methods very common, but not limited to, web development

http and ftp are not necessarily used inside a GUI like a browser or filezilla, there are other tools using same protocols for communication between devices.

Some of those tools are text-based or command line browsers, the http cli-program in linux.

There is also related software to download webpages, for example wget or curl.

File Transfer Protocol (FTP)

  • Use port 21 to be secured?
  • exchange files with ftp-server. `ftp` on command line or gui clients.
  • Apart from port 21, it's not a secure way to transfer files.
Example
On a browser's address bar: ftp://address
cli: ftp adress
Filezilla: write adress in search box.

Without a secure connection hackers can see almost anything transferred through the network (particularly important in open networks where many users are connected). So this is not a secure method to buy things, log into networks etc. but we still can look up the bus.

For sensible data SFTP should be used instead. SFTP uses secure shell, encrypts the data. Port 22.

In any case, we need the IP, a user and a password.

HyperText Transfer Protocol (HTTP)

HyperText Transfer Protocol Access public files of a server. Does not ask for passwords. It's possible (very easy) to download the html, and any files in the public folder.

  • On a Browser Address Bar: http://www.domainName.subDomain
  • Using wget and curl: wget/curl [options] webaddress; or GET, POST, HTTP + webadress
