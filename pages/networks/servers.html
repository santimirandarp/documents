Servers

Servers are computers offering services to other computers. They both must be on a network. The consumer computer is the Client.

There are specific computers used as servers, but our own computer can be converted into a server.

Web Browsers like Mozilla Firefox are HTTP-server (or web) clients. We can access FTP servers using Filezilla, or any other FTP client. We can access SMB servers in a similar way, the server could be set up using SMB. We can also access SSH servers, etc.

In general, we connect to particular types of servers using particular clients.

One computer can be multiple servers, it's not just one server on each computer.

Hardware

Normally servers use Xeon processors, redundant power supplies (RSP) so if one dies the system keeps running (ideally it runs 24/7). We can take out and replace it. Redundant array of inexpensive disks (RAID). The computer maintains more than one copy of the data, using many hard drives, if one cracks down, we can replace it, and everything is rebuild. Error-code Correction (ECC) RAM runs a check of the RAM.

Software

The server type correlates with a program, not with a machine. A machine could be FTP, SMTP, SSH and HTTP server, for example. Security or Authentication server, so we log in and gain access to a subset of directories in the server. Apache, Nginx are for web servers, SQL for databases.

DNS server

It's like phone book (we know the name, get the number). DNS resolves domain names to IPs. A domain name could be google.com. If the domain does not exit, we get an error.

To access any device in the world wide web (network) we need to be connected to WAN. This is done using a Router/Switch. Which in turn needs to be configured. The IP of the router is retrieved by using: ip route in linux, and similar in windows.(any device on a network is assigned an IP).

Proxy Server

It's a computer that asks for a webpage for us, so our IP remains hidden. Instead of asking for an address straight to the web-server, we ask to the proxy and this (with its IP) asks for the webpage or data.

The benefit is surfing the web anonymously. One problem, it doesn't encrypt data, so in the worse scenario user/pass could be detected by hackers.
