CSS basics

  • Best practice: css in external file.

HTML5 controls a webpage's structure and semantics. CSS controls how the page looks.

Syntaxis: selector {css rules}. The selector points to an HTML element.

How to Write CSS

Syntaxis 

selector {property: value;}

Selectors: .class, #id, tag, attribute value (forms).

Multiple Classes 

<div class= 'className1 className2'> Some Random Text<div>

Brief

Elements come with custom styling (user agent stylesheet), or inherit a property, or can be styled using any selector. Inherited props can be overwritten, assigned props fight with each other and the more specific wins. If specificity is the same, then source order defines.

Inheritance and Overwriting

Some properties are inherited (passed on from parents to children). An example is color (with the exception of anchors), fonts (with exception of input forms), text-align, background. We expect those to be inherited. On the other hand width, margin, border, padding are not inherited.

Clashes

An elements styling then, can be set on its own (by us or agent's stylesheet) or inherited. If we reassing an inherited property, as we expect, it gets overwritten. 

        body{color:black;}

        h1 {color:red;} //overwrittes black
        
        h1 span {color:green;} //overwrittes red
        
        p, h3, h4 {//inherit color from body!} //inherit

What happens when 2 rules refer to the same HTML element? The algorythm decides which one to apply. How?

Two concepts matter here:

  • Specificity: The Rule of 4 places
  • Source Order (aka Cascade): Last rule defined applies.

CSS and the Browser

Browsers ignore chunks of code they do not understand either properties-values or blocks (if the selector is not understood). Dom is different to View Source. The browser may correct errors, and modify the DOM if js is being applied. This error correction might be different in diff browsers. So it's a good idea to validate the code in html and css validator.

We can select inspect elements (selecting on the page) or in the DOM. The css rules appear on the right, crossed over rules might mean different things: overwritten rule, rule not understood by the browser, doesn't exist, etc.
