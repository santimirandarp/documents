The Cascade or 'Who Wins'?

HTML5 controls a webpage's structure and semantics. CSS controls how the page looks.

It has its own syntaxis: selector {css rules}. The selector points to an HTML element.

Inheritance & Overwritting

Some properties are inherited (passed on from parents to children). This is another way of styling. If children are re-styled on the same property -than the parent, the property gets overwritten.

width, margin, border, padding are not inherited.

color, font-family, text-align, background are inherited.

Clashes

An elements styling then, can be set on it own or inherited.

        body{color:black;}

        h1 {color:red;} //overwrittes black
        
        h1 span {color:green;} //overwrittes red
        
        .h2bgcolor {background-color:grey;} // set on its own

        p, h3, h4 {//inherit color from body!} //inherit

If they refer to the same element, the algorithm decides which one to apply.

Two concepts matter here:

  • Specificity: The Rule of 4 places
  • Source Order: Last rule defined applies.
