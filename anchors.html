Anchors' Attributes

An anchor tag contains 4 interesting and frequent attributes:

  • href for hypertext reference,
  • target:[self|blank]
  • title w supplementary info about the link.
  • download: <a href='file.extension' download='nicename.ext'> good text <a>

Download only works for same origin URLs, this is, relative paths

[title] attribute is available in many other elements. Screenreaders, touch screens, keyboard-only users, text-only browsers do not see this feature as it requires hovering.

Clean URL

Browsers look for index.html automatically. Indeed, the path can be contracted to the folder. Bonus: the filename doesn't appears in the browser's URL.

Linking to a section of any HTML page

Elements with an id as in 

<section id="idName"> content <section/>
can be linked in an anchor's hypertext reference attribute too, just using href='/path/to/file#idName'. /path/to/file should be omitted for a section in the same document (as we are already there).

URLs enconding

URLs have to be ASCII encoded, otherwise, the browsers handle the conversion using %0F, hexadecimal chars. For example the space is converted to %20 or to a + sign too.

[href]='mailto:'
