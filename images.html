Images

Loading, Best Practices, Theory

Loading

In the most straightforward approach images can be loaded using

<img src='path/to/images/sourceImage.format'>

Best Practices

store on an /images folder & use descriptive_filename
for SEO purposes.
on the same server (avoid using external urls).
avoid links breaks or modified images.
A DNS search + GET takes more time to load the image.
Use alt attribute. Text is useful for users with:
impaired vision
turned off images (ex. in many phones)
text-only browsers as Lynx
search engines
decorative images use background-image.
they dont add to the content and do not need to be described
if you really need a decorative <img> use alt=''
If an external image is used, be it on the public domain. Two useful resources:
unsplash.it/h/w
picsum.photos/h/w/?random
width and height
Use always to avoid flickering and reserve space

About the alt attribute

  • alt attribute is required for validation (use html validator)
  • images inside links add border:0 (to prevent IE9 and earlier of displaying a border)
  • Minify the images to reduce load time + data-transfer.

Loading Time, Resolution, Retina Displays

In the bare minimum and image loading should look like this: <img alt='description' width='300' height='400' src='path/to/pic'/&bg;

Common computer-screen resolutions are 72dpi

An image should be displayed on its natural size (area covered by its number of pixels). Why? Too big sizes make it grainy, too small (if too small) waste bandwidth and increase load time.

It's ok to strech/shrink but avoiding extremes. srcset attribute allows to include many different resolutions and sizes.

Example Case

Let's say we want to upload a 1500px square image, and make it available for retina displays and responsive.

We could .image {width:100%} for responsiveness. But yet it loses resolution when stretching out or wastes bandwidth when shrinking in. We can add srcset and upload diff sizes for different viewports. So it still streches or shrinks but not that much.

Control Of Backgrounds

This is an interesting approach to control the display of a background for different devices. It uses min-device-width instead of device-width, the difference being that min-device-width doesnt pay attention to the viewport. Both can be used though.
