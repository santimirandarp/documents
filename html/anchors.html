Anchors' Attributes

Frequent attributes:

  • href for hypertext reference,
  • target:[self|_blank]
  • title w supplementary info about the link.
  • download: <a href='file.extension' download='nicename.ext'> good text <a>

Extended Discussion

href accepts relative paths (relative to index.html's folder) and absolute paths (full url to specify a computer plus a path to the file.) HTTP protocol allows to access files in a different computer called server, plus some post processing.

target has two main values: _self or _blank. self opens up the reference on the same window, blank does it on a new tab. download only works for same origin URLs, this is, relative paths (in the same computer i.e server.)

[title] attribute is available in many other elements. Screenreaders, touch-screens, keyboard-only users, text-only browsers do not see this feature as it requires hovering.

A concrete example

A full example using relative paths to about.html page would be:

<a href="./about.html" title="Opens a new tab on the about page for this website" target="_blank">Go to About Page</a>

A full example using absolute paths would be:

<a href="http://cdimage.ubuntu.com/lubuntu/releases/20.04/release/lubuntu-20.04-desktop-amd64.iso" title=" lubuntu 20.04 will start downloading with the name: lubuntu.iso" target="_self" download="lubuntu.iso">Download Lubuntu 20.04 64-bits iso (opens on current window).</a>

How to write nice and useful URLs

Clean URL

Would you like to display baseurl/about instead of baseurl/about.html? Then read on.

Example

Suppose on the project folder there are all html files: index.html, about.html, contact.html, whatever.html. Now we make new directories named: about, contact, whatever. Next we put about.html under about directory and rename the file about.html as index.html. Last, we follow the same procedure with every file.

On the main index.html file we link to about as href="about" and analogous for the remaining files. When you click on about the browser will display a nicer url.

Linking to a section of any HTML page

Elements with an id as in 

<section id="idName"> content <section/>
can be linked in an anchor's hyper reference attribute too, just using href='/path/to/file#idName'. /path/to/file should be omitted for a section in the same document (as we are already there.)

URLs encoding

URLs have to be ASCII encoded, otherwise, the browser handles the conversion using %0F, hexadecimal chars. For example the space is converted to %20 or to a + sign too.

In this sense, you don't want filenames with spaces or non-ascii characters. If you do, the browser will modify it and probably won't look good or trusty. And for sure it won't look professional.
