Background

Background common props and vals

  • background-color
  • background-image (by default on top of the color). Values: url(pathtoimage), gradients like radial-gradient(vals). As a default, because of background-size, it will cover the element.
  • background-size: cover| val1 val2 | keywords. We can give it pixels, %, etc. Or one keyword: contain or cover. (They are self explanatory).
  • background position: left offset top offset| many options (read docs if needed). Tells where to position the image in the defined box space. We can add offsets. Its a shorthand for background-position-x and bgp-y.
  • background-origin: tells where to start 'drawing' the image. Defines the box space.
  • background-repeat: repeat (x and y) | no-repeat|repeat-x|repeat-y
  • background-attachment:scroll|fixed|local (experimental) scrolls with the page, then you see just the initial piece drawn. fixed it is fixed to the viewport, so it's possible to give it some movement.

Accessibility Implications in backgrounds images and content images

  • Text must be legible (contrast)
  • Always provide a background-color as a fallback.
  • Do not put content into background (screen readers).

Multiple Layers is enabled by comma separating values. Last image goes on bottom.

Control Of Backgrounds
